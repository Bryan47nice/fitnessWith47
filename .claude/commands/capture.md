# /capture

快速將 idea 或 bug 歸檔到 overnight backlog，不需要知道格式。

**用法：** `/capture 描述你的想法或問題`

---

## 執行步驟

### Step 1 — 理解描述

分析用戶輸入，判斷：
- **類型**：bug（現有功能壞掉）/ feature（全新功能）/ enhancement（改善現有功能）
- **複雜度**（依以下標準）：

| 複雜度 | 條件 |
|--------|------|
| **簡單** | 單一元件改動、無新資料結構、無 Firestore schema 異動、預估 < 50 行 |
| **複雜** | 多元件協作、需新資料結構、新 Firestore collection、顯著 UX 流程變更 |

### Step 2 — 判斷優先度

若描述中有明確優先度意圖（「很急」「小事」「之後再說」）則採用；
否則依類型判斷：
- bug → 預設 [高]
- feature / enhancement → 預設 [中]，並說明理由讓用戶確認

### Step 3 — 寫入 backlog

讀取 `docs/overnight-backlog.md`，在 **Pending 區塊最前面**（若為高優先）或適當位置插入：

**簡單：**
```
- ⏳ [高/中/低] {一句話描述功能或 bug}
```

**複雜：**
```
- 📋 [需規劃] {一句話描述功能}（複雜原因：{說明為何複雜}）
```

### Step 4 — 回報結果

```
✅ 已加入 backlog

類型：{bug / feature / enhancement}
優先度：{高/中/低}
複雜度：{簡單 → 今晚直接實作 / 複雜 → 今晚先出規劃，明早供你核准}

目前 Pending 清單共 N 項，此項排在第 M 位。
```

若優先度是自行判斷的，說明理由並詢問「這個優先度 OK 嗎？」，讓用戶可以調整。
